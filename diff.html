What Changed Conceptually

main

Built-site assignment stops at delivery

  • assign_built_site links a prebuilt site to an attendee.
  • Read-only mode is controlled by a simple READ_ONLY env flag.
  • Payment intent has no site renewal context.
  • Admin can edit built-site metadata but not operate renewal state.
this branch

Built-site assignment creates a renewable asset

  • Assignment provisions a renewal token and an initial deadline.
  • Read-only mode is driven by READ_ONLY_FROM plus warning window logic.
  • CheckoutIntent.siteToken threads renewal identity through Stripe/webhooks.
  • Admin can provision, rotate, bump, override, and re-sync renewal state.

New Admin Operations

  • provision-renewal: create token, deadline, and edge secrets after a qualifying tier exists.
  • rotate-renewal-token: swap public link while preserving current deadline.
  • bump-deadline: add up to 120 months from current deadline or now.
  • override-deadline: set an explicit YYYY-MM-DDT23:59:59Z cutoff.
  • re-sync-deadline: re-push stored cutoff and renewal URL to Bunny.

Payment Semantics

  • Renewal checkout is still the normal ticket picker and Stripe path.
  • Only qualifying tiers are shown: months_per_unit > 0, hidden, purchase-only.
  • Multi-tier renewal is supported: quantity times months per unit per item.
  • Unknown or missing token returns 404; no tier renders a renewal-unavailable page.

Operational Notes

  • scripts/build-site.ts provisions from a freshly built local edge bundle.

Review Lens For Maintainers

Invariants introduced

  • A built site is provisioned for renewals iff renewalTokenIndex !== null.
  • Raw token is stored only inside encrypted site_data; lookup uses HMAC index.
  • Assigned-site events require initial_site_months > 0.
  • Renewal tier events must be hidden and purchase-only.

Failure paths now matter

  • Bunny secret push can fail after token generation; token is retained so admin can re-sync.
  • Renewal payment can succeed while deadline push fails; error logging and ntfy are used.
  • Expired renewal extends from now, not from the stale past cutoff.
  • Multiple renewal tier events are valid; customer picks the tier at renew time.
